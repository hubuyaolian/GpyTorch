# 《AI进化史：从感知机到Transformer的觉醒之路》总体大纲

这份路线图完全摒弃了"枯燥讲API"的传统模式。我们将带你重走一遍深度学习的历史：了解前人遇到了什么**无法解决的痛点**，从而发明了哪些**改变历史的名词与架构**。

在代码层面，我们将主打**"逻辑复现"**。不再深陷极其冗长、让人眼花缭乱的纯底层数学推导，而是用最精简、直白的 PyTorch 代码（结合基础张量运算和核心模块），一针见血地还原这些伟大架构的本质。

## ❄️ 第一阶段：AI的初春与寒冬（第1周）

**【历史背景】**：五十年代，人们以为只要把"神经元"用数学公式表达出来（感知机），AI 就能很快拥有人类智能。但很快，人们发现它连最基本的逻辑判断都做不对。

| 天 | 主题 | 核心内容 |
|----|------|----------|
| Day01 | 张量——深度学习的通用货币 | 张量基础、矩阵乘法、广播机制、感知机的计算预演 |
| Day02 | 生物神经元与感知机的诞生 | 生物神经元映射到感知机、搭建感知机骨架 |
| Day03 | 权重与偏置——赋予神经元灵魂 | 权重/偏置的直觉、前向传播完整实现 |
| Day04 | 线性分类边界——感知机的几何直觉 | 决策边界可视化、AND/OR 线性可分验证 |
| Day05 | XOR危机——AI第一次寒冬的代码现场 | XOR 数学证明不可解、10000次随机尝试全部失败 |

*   **核心名词**：感知机（Perceptron）、权重（Weight）、偏置（Bias）、线性分类边界。
*   **解决的痛点**：早期的逻辑回归与简单的线性分类问题。
*   **逻辑复现代码**：用最基础的张量运算构建单层感知机。直观演示并证明它为什么无法解决"异或（XOR）问题"（线性不可分），重现导致AI第一次历史寒冬的代码现场。

## ☀️ 第二阶段：破局与复兴（第2周）

**【历史背景】**：被 XOR 问题折磨了几十年后，人们终于找到了破局之法：加深网络（隐藏层）和一套神奇的算账法则（反向传播）。

| 天 | 主题 | 核心内容 |
|----|------|----------|
| Day01 | 隐藏层——打破线性边界的魔法 | 隐藏层如何解决 XOR、空间变换直觉 |
| Day02 | 非线性激活函数——Sigmoid与ReLU | 阶跃函数梯度困境、Sigmoid/ReLU/Tanh 对比（**梯度消失第 1 次登场**：激活函数压缩梯度 → 换 ReLU 解决） |
| Day03 | 损失函数与梯度下降——模型的吐槽大会 | MSE/BCE/CE 损失函数、SGD/Adam 优化器、梯度下降可视化 |
| Day04 | 反向传播的直觉——链式法则的代码实现 | 信用分配、链式法则、手写两层 MLP 反向传播 |
| Day05 | 手写MLP——用代码解决XOR问题 | PyTorch autograd、完整训练循环、MLP 解决 XOR |

*   **核心名词**：多层感知机（MLP）、隐藏层（Hidden Layer）、非线性激活（Sigmoid/ReLU）、反向传播（Backpropagation）、损失函数、梯度下降。
*   **解决的痛点**：隐藏层解决了线性不可分问题；反向传播解决了"如何将最终误差的责任分摊给中间隐藏层"（信用分配问题）。
*   **逻辑复现代码**：用简练的代码实现一个带有一层隐藏层的微型网络，演示反向传播是如何更新隐藏层权重的，成功解决 XOR 问题。
*   **状态**：✅ 已有 Notebook

## 👁️ 第三阶段：视觉的征服（第3周）

**【历史背景】**：给模型看图片时，全连接网络的参数瞬间爆炸，且一旦把图片里的猫往旁边挪一点，它就不认识了。于是，灵感来源于人类视觉皮层的 CNN 诞生了。

|----|------|----------|
| Day01 | 卷积的诞生——让机器学会看图 | 卷积运算、边缘检测卷积核、Sobel 算子、局部感受野、权值共享 |
| Day02 | 卷积层——让机器自己学习特征 | nn.Conv2d、训练卷积核学习特征、特征图可视化 |
| Day03 | 池化——信息的浓缩与抽象 | MaxPool/AvgPool、平移不变性、全局池化 |
| Day04 | 构建CNN——手写LeNet-5 | 完整 LeNet-5 架构搭建、MNIST 训练 |
| Day05 | 梯度消失——深度网络的老年痴呆 | 深层 vs 浅层网络对比、Sigmoid 梯度衰减实证（**梯度消失第 2 次登场**：网络深度导致梯度衰减 → 引出 ResNet） |

*   **核心名词**：感受野（Receptive Field）、卷积核（Kernel）、权值共享、特征图（Feature Map）、池化（Pooling）、梯度消失。
*   **解决的痛点**：全连接层处理图像导致参数量爆炸；图像平移后模型无法识别。
*   **逻辑复现代码**：用基础张量运算手写 2D 卷积（滑动窗口 + 逐元素相乘 + 求和），可视化边缘检测效果；搭建 LeNet-5 并在 MNIST 上训练验证。

## 🔧 第四阶段：训练深度网络的秘籍（第4周）

**【历史背景】**：上周我们发现，深层网络的梯度会像传话一样越传越小——这只是训练深层网络的众多噩梦之一。即使梯度能正常传播，训练过程还面临收敛太慢、过拟合严重、中间层分布漂移等挑战。2012-2015 年间，一系列训练技巧的发明才让深度学习真正从实验室走向现实。

| 天 | 主题 | 核心内容 |
|----|------|----------|
| Day01 | 优化器进化论——从SGD到Adam | 动量、自适应学习率、Adam 的完整实现 |
| Day02 | Dropout——训练时的随机失忆 | 随机失活、集成学习直觉、过拟合 vs 欠拟合 |
| Day03 | BatchNorm——给每层输入重新站队 | 内部协变量偏移、归一化、可学习参数 γ/β |
| Day04 | 学习率调度——下山节奏的把控 | StepLR、CosineAnnealing、Warmup |
| Day05 | 完整训练流程——组合技实战 | 将上述技巧组合到 CNN 中、对比实验 |

*   **核心名词**：动量（Momentum）、自适应学习率、Dropout、批归一化（BatchNorm）、学习率调度。
*   **解决的痛点**：收敛慢、过拟合、内部协变量偏移、训练不稳定。
*   **逻辑复现代码**：手写带动量的 SGD（不依赖 optimizer）；手写 BatchNorm 前向传播（均值/方差归一化 + γ/β 缩放）；对比有无 Dropout/BatchNorm 的训练曲线。

## 🏗️ 第五阶段：CNN进化论——从LeNet到现代（第5周）

**【历史背景】**：LeNet-5 只有 5 层，准确率有限。人们开始追问：能不能堆更多层？VGG 用统一的 3x3 卷积堆到了 16-19 层，GoogLeNet 用多尺度并行结构拿到了 ImageNet 冠军。但层数继续增加时，梯度消失再次出现——这为 ResNet 埋下了伏笔。

| 天 | 主题 | 核心内容 |
|----|------|----------|
| Day01 | VGG——用3x3统一天下 | 小卷积核堆叠、通道数递增、参数量分析 |
| Day02 | 1x1卷积——通道数的魔法压缩 | 升维/降维、跨通道信息融合、计算量优化 |
| Day03 | GoogLeNet——多尺度并行侦察 | Inception 模块、1x1 卷积降维、多分支结构 |
| Day04 | 全局平均池化——告别全连接层 | GAP 的优雅、参数量对比、CAM 可视化的基础 |
| Day05 | 深度的瓶颈——梯度消失重现 | 20层 vs 50层 PlainNet 对比、退化问题引出 |

*   **核心名词**：VGG、Inception、1x1 卷积、全局平均池化（GAP）、退化问题（Degradation）。
*   **解决的痛点**：如何设计更深的网络架构；梯度消失问题的再次出现。
*   **逻辑复现代码**：用矩阵乘法实现 1x1 卷积（展示跨通道信息融合）；手写 Inception 模块（多分支并行 + 拼接）；对比 PlainNet 20层 vs 50层 的训练曲线，重现退化问题。

**【历史背景】**：当人们以为"网络越深越好"时，发现超过20层后，模型不仅没变聪明，反而变得更蠢了（退化问题）。何恺明带着 ResNet 横空出世，一举将网络推向了152层。

## ⏳ 第七阶段：记忆的诞生（第7周）

**【历史背景】**：ResNet 让视觉问题基本解决了——再深的网络也能训练。但现实中还有一类数据，CNN 完全搞不定：有时间顺序的数据——语言、语音、股票走势……不管是 MLP 还是 CNN，它们都是"看一眼算一次"，看完就忘。遇到需要理解上下文的句子，它们就束手无策了。我们需要赋予机器"记忆"。

| 天 | 主题 | 核心内容 |
|----|------|----------|
| Day01 | 金鱼记忆——为什么前馈网络记不住 | 序列数据的特殊性、时序依赖 |
| Day02 | 隐藏状态——RNN边看边记的秘密 | RNN 单元、隐藏状态传递、手写 RNN |
| Day03 | 梯度消失（续）——RNN的致命缺陷 | 时间维度的梯度爆炸/消失、BPTT（**梯度消失第 3 次登场**：时间维度的梯度衰减 → 引出 LSTM 门控机制） |
| Day04 | LSTM——三个门的魔法 | 遗忘门/输入门/输出门、细胞状态 |
| Day05 | RNN vs LSTM 终极对决 | 长序列对比实验、门控机制可视化 |

*   **核心名词**：循环神经网络（RNN）、隐藏状态（Hidden State）、长短期记忆（LSTM）、门控机制。
*   **解决的痛点**：传统网络无法处理变长序列；基础 RNN 的长距离依赖问题，LSTM 通过门控解决。
*   **逻辑复现代码**：手写 RNN 单元（隐藏状态的矩阵乘法 + 传递），观察一句话从头到尾的记忆流动；手写 LSTM 三个门（遗忘/输入/输出），对比 RNN vs LSTM 在长序列上的梯度传播。

## 🚀 第九阶段：注意力时代——Transformer的觉醒（第9周）

## 🎯 第十二阶段：对齐之路——让模型听话（第12周）

**【历史背景】**：GPT 虽然强大，但它只是一个"文字接龙机器"，不懂什么是"有帮助"、什么是"安全"。RLHF（基于人类反馈的强化学习）教会了模型理解人类偏好。而 DPO 提出了一种更简洁的替代方案——不用强化学习也能对齐。

| 天 | 主题 | 核心内容 |
|----|------|----------|
| Day01 | 对齐问题——为什么模型不听话 | 语言模型的目标 vs 人类期望的差距 |
| Day02 | 指令微调——教模型听懂人话 | Instruction Tuning、对话数据格式 |
| Day03 | 奖励模型——给回答打分 | 偏好数据、Bradley-Terry 模型、奖励函数 |
| Day04 | RLHF全流程——从评分到优化 | PPO 算法直觉、策略梯度、KL 惩罚 |
| Day05 | DPO——不用强化学习也能对齐 | Direct Preference Optimization、与 RLHF 对比 |

*   **核心名词**：RLHF、奖励模型（Reward Model）、PPO、DPO、对齐（Alignment）。
*   **解决的痛点**：预训练模型不理解人类偏好；需要对齐才能安全使用。
*   **逻辑复现代码**：手写 Bradley-Terry 奖励模型（偏好对的 sigmoid 损失）；用代码演示 PPO 的 clip 机制（限制策略更新幅度）；对比 RLHF vs DPO 的训练流程差异。
*   **状态**：⚠️ 部分已有（现有 Week15-16 内容需重组）

## 🧠 第十三阶段：思维与工具——让模型学会思考（第13周）

**【历史背景】**：即使是 ChatGPT，面对复杂数学题也会犯蠢。人们发现，让模型"一步一步想"（Chain of Thought）能大幅提升推理能力。更进一步，给模型接入工具（计算器、搜索引擎），它的能力边界被彻底打破。

| 天 | 主题 | 核心内容 |
|----|------|----------|
| Day01 | 提示工程——与模型对话的艺术 | 提示设计原则、系统提示、上下文学习、Few-shot |
| Day02 | 思维链——让模型一步步想 | CoT 的惊人效果、Few-shot CoT、Zero-shot CoT |
| Day03 | 工具使用——给模型装上手脚 | Function Calling、工具编排 |
| Day04 | ReAct——边想边做的行动框架 | 推理-行动交替、观察-思考-行动循环、Agent 雏形 |
| Day05 | 推理模型——会思考的AI | o1/DeepSeek-R1 类模型、思维链蒸馏、推理时间计算 |

*   **核心名词**：Chain of Thought（CoT）、Tool Use、Function Calling、ReAct、Prompt Engineering、推理模型（Reasoning Model）。
*   **解决的痛点**：模型推理能力不足；模型无法访问外部信息和工具。
*   **逻辑复现代码**：用代码对比 Zero-shot vs Few-shot vs CoT 的推理准确率；手写 Function Calling 的工具调度逻辑（解析 JSON → 调用工具 → 注入结果）；实现一个最小 ReAct 循环（思考→行动→观察→思考…）。

## 🤖 第十四阶段：Agent时代——从工具到自主（第14周）

**【历史背景】**：当模型能思考、能使用工具、能记住上下文时，它就不再是"聊天机器人"了——它变成了 Agent。一个能自主规划、执行、反思的智能体。

| 天 | 主题 | 核心内容 |
|----|------|----------|
| Day01 | Agent架构——大脑+手脚+记忆 | 感知-规划-行动循环、Agent 与传统程序的区别 |
| Day02 | 规划模块——把大任务拆成小步骤 | 任务分解、层次化规划、自我反思 |
| Day03 | 记忆模块——短期与长期记忆 | 工作记忆、向量数据库、检索增强 |
| Day04 | 反思模块——从失败中学习 | 自我评估、经验总结、策略迭代 |
| Day05 | 多Agent——团队协作的力量 | 角色分工、通信协议、冲突解决 |

*   **核心名词**：Agent、规划（Planning）、记忆（Memory）、反思（Reflection）、多智能体（Multi-Agent）。
*   **解决的痛点**：从"回答问题"到"自主完成任务"的范式转变。
*   **逻辑复现代码**：搭建一个最小 Agent 框架（LLM + 工具调用 + 简单记忆）；手写任务分解逻辑（大目标 → 子任务列表 → 逐步执行）；实现带反思的 Agent 循环（执行 → 评估 → 调整策略 → 重试）。

## 🎓 收官：从感知机到 Agent 的觉醒之路（第14周后）

**【回顾三条主线】**：

1. **信息流动**：单神经元 → MLP → CNN → RNN → Transformer → Agent（从"看一眼"到"自主行动"）
2. **参数效率**：全连接 → 权值共享 → 残差连接 → 注意力（从"暴力堆参数"到"精准分配计算"）
3. **记忆机制**：无记忆 → 隐藏状态 → 门控 → 全局注意力 → 外部记忆（从"金鱼记忆"到"长期记忆"）

**【实战收官】**：搭建一个完整的小型 Agent 项目——整合 LLM 推理、工具调用、记忆检索、任务规划，体验从"聊天机器人"到"自主智能体"的最后一公里。

**【展望】**：从 Agent 到 AGI，我们走到了哪里？当前的局限与未来的可能。
