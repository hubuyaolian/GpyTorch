# 🛑 Day05: 提示词工程与防幻觉生成 (RAG 收官)

**【真实工作痛点】**
“我辛辛苦苦把最正确的公司考勤规章送到了大模型嘴边。结果用户问：‘迟到了怎么办？’，大模型觉得文本太少不过瘾，竟然结合它在网上看的资料，瞎编了一句：‘除了扣钱，还要找总经理罚站检讨。’ 老板看到测试结果脸都绿了！”

在 RAG 的最后一环，大模型的“发散思维”反而成了毒药。如果不施加极其强硬的**系统提示词 (System Prompt) 约束**，大模型一定会产生**幻觉 (Hallucination)**。

---

## 步骤 1：确立最高铁律 (System Prompt)

你需要通过系统消息，给大模型带上“紧箍咒”。切断它调用自身已有世界知识的冲动。

```python
system_instruction = """
你是一个严格依据事实回答问题的企业知识库助手。

【最高铁律】
1. 你必须、且只能使用下方提供的 [参考资料] 来回答用户的问题。
2. 如果 [参考资料] 中没有包含足够的信息，你必须直接回复：'抱歉，当前知识库中未找到相关内容。'
3. 绝对不允许根据你自己的常识进行任何编造、推测或扩写。
4. 回答时，必须在句末引用你依据的参考资料编号（如：[片段1]）。
"""
# 这段话必须作为 role="system" 发送给 API。
```

---

## 步骤 2：组装上下文砖块 (Context Assembly)

拿到了 Day04 Rerank 过后的 Top-K 文档，我们不能生硬地把它们扔在一堆，而是要排好版，让大模型能分清哪段是哪段。

```python
# 假设这是查出来的最准的两条资料
retrieved_contexts = [
    "员工手册第4章：迟到单次扣除全勤奖 50 元。",
    "考勤系统说明：忘记打卡需在 3 日内提交补卡流程。"
]

context_block = ""
# 给每个片段打上编号标头
for i, context in enumerate(retrieved_contexts):
    context_block += f"【片段 {i+1}】:\n{context}\n\n"
    
# 组装后的 context_block 变成了易于阅读的带编号结构：
# 【片段 1】:
# 员工手册第4章...
# 【片段 2】:
# 考勤系统说明...
```

---

## 步骤 3：拼接最终的用户消息 (User Message)

我们要用清晰的分割线（比如 `---` ），把“知识库内容”和“用户的问题”物理隔离开来。防止大模型搞混。

```python
user_query = "我早上迟到了，会有什么后果？"

final_user_message = f"""
以下是为你提供的 [参考资料]：
---------------------
{context_block}
---------------------

请严格根据上述 [参考资料]，回答以下问题：
用户问题：【{user_query}】
"""

print(final_user_message)
```

**最后一步，将 `system_instruction` 和 `final_user_message` 喂给模型，RAG 链路完美闭环！**
