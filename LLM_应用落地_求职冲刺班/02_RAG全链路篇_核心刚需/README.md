# 🎯 02. RAG 全链路篇：核心刚需

> **学习原则：** 这是原课程大纲中严重缺失，但目前**企业需求量最大**的部分。必须精读，必须动手踩坑！

## 🏗️ 核心实战目录：

### 1. 文档解析与切分 (Chunking) 的艺术
* 坑点：切分太粗暴（一句完整的话被切成两截），导致召回彻底废掉。
* 实战：按标点符号切分 vs 按语义段落切分 vs 滑动窗口（Overlap 重叠设置）。

### 2. 向量化 (Embedding) 的选择与降维
* 实战：对比不同开源 Embedding 模型（如 BGE、m3e）在中文上的表现。

### 3. 向量数据库选型与踩坑
* 实战：放弃重量级框架，从最底层的 `FAISS` 和轻量级的 `Chroma` 开始，理解相似度检索（Cosine Similarity, L2）的原理。

### 4. 检索优化：提升召回率的命门
* 痛点：只返回 Top3 答案却在第 5 段？召回噪音太多带偏模型？
* 实战：
  * **混合检索**：向量检索 (语义) + BM25 (关键词)。
  * **重排序 (Rerank)**：引入交叉编码器 (Cross-Encoder) 对粗排结果进行二次打分。

### 5. 提示词与生成 (Generation)
* 实战：如何写死 Prompt 限制大模型：“仅使用提供的上下文回答，如果不知道请回答'不知道'”，从而**解决幻觉问题**。
