# ⚡ 04. 工程调优与多模态篇

> **学习原则：** 解决“能不能用”之后，解决“好不好用”的问题。这是区分实习生和资深应用开发的分水岭。

## 🏗️ 核心实战目录：

### 1. 响应极速化：Streaming (流式输出)
* 痛点：用户等 3 秒才出字，体验极差。
* 实战：手撕 SSE (Server-Sent Events) 前后端链路。计算并优化 TTFT (Time To First Token，首字响应时间)，把 3 秒压到 0.5 秒以内。

### 2. 结构化输出的命门：JSON 强制化
* 实战：做业务最怕大模型乱回话。学习如何使用模型的 JSON Mode，或者通过 Prompt Engineering 结合 Pydantic 强制让大模型吐出完全合规的结构化数据，以供代码继续流转。

### 3. 多模态实战 (Vision & Audio)
* 实战：
  * **图像理解**：结合 Vision 大模型（如 GPT-4o 或 Qwen-VL），上传复杂的财务发票或课本图片，提取知识点。
  * **语音交互**：接入语音转文本 (ASR) 与文本转语音 (TTS) API，做一个语音对讲 Agent。
